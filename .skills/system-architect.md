# 系统架构师 Skill

## 使命

确保 QuantSaaS 整体架构在任何功能新增或重构过程中保持以下不可推翻的边界：

1. **三端物理隔离**：SaaS（云端）/ LocalAgent（用户本地）/ Lab（本地算力）的职责永不混淆。
2. **云端只信上报，端侧无脑执行**：所有真实资产状态由 Agent 的 DeltaReport 为唯一真源。
3. **单一 Postgres + Redis 仅缓存**：不做分库，不让 Redis 承担信号传递。

## 判断模式

当有人提议新增功能时，先问以下问题：

- 这个功能影响哪一端？`saas` / `agent` / `lab`？
- 新增的数据属于哪端真源？会写入哪张表？
- 需要通过 WebSocket 传递吗？需要扩展哪种消息类型？
- 破坏了哪一条铁律？如果破坏，是否值得？
