## 核心准则：进化的物理意义
GA 算法的目的**绝不是为了寻找历史最高收益率**（那只是过拟合的数字游戏），而是要在 10 年的极端市场环境（暴涨、阴跌、连环爆仓）中，寻找一个能够完美平衡**“高胜率决策”**与**“极低摩擦成本”**的数学极值点。

回测与进化模块是独立于实盘执行端（Live Agent）的实验室。它必须绝对冷酷，以“全悲观”的假设进行压力测试。

## 第一篇章：基因编码与染色体结构 (Chromosome Design)
在系统中，每一个“交易策略个体”（Agent）都携带一串由浮点数和整型构成的基因序列。基因被严格划分为两大功能组：

### 1. 宏观基因组 (Macro Genes - “老农定投模型”)
决定系统在长周期（日线/4H）级别的资产配置与建仓节奏；**全部进入 LunarGenome，由 GA 交叉与突变演化**。

+ **MaxDCAMonths**（最大定投月数）：限制宏观子弹消耗速度，防止在阴跌中过早打光储备。
+ **BetaThreshold**（大盘偏离度阈值）：决定触发滴灌或深跌加仓的宏观价格偏离门槛（个体“头铁”程度）。
+ **MoonPhasePressure**（月相压力）：接近满月时放大定投节奏的强度系数。
+ **DeadlineForcePct**、**GCThresholdMonths**、**GCMaxRatio**：耐心耗尽时的强制动用比例、长期闲置回收门槛与上限（宏观账本纪律）。
+ Timing 结构基因：**TMacro / TMicro / TDeadline / EMAAnchor**（与宏观统计窗口和节奏相关，随基因组一起进化）。

### 2. 微观基因组 (Micro Genes - “狙击手动量模型”)
决定系统在短周期（1m/5m）级别的 PDE（偏微分方程）敏感度与开火权限。

+ **PDE 动力学权重**：
    - $ kp $** (位置权重)**：当前价格距离均值的引力系数。
    - $ kv $** (速度权重)**：对一阶导数（价格变化率）的敏感度。
    - $ ka $** (加速度权重)**：对二阶导数（趋势反转与极值点）的敏感度，直接决定高位逃顶的锐度。
+ `**MinTradeThreshold**`** (最小交易偏差阈值)**：**极其关键的基因**。决定当前目标权重与实际权重偏差达到多少时才允许扣动扳机，用于过滤震荡市的无效交易。
+ `**MicroReserveRate**`** (微观每次调拨率)**：每次开火时，动用可用资金的百分比，决定弹药的倾泻速度。

## 第二篇章：适应度函数 (Fitness Function & Scoring)
适应度函数是 GA 算法的“死神”。一个基因序列是存活还是被淘汰，完全由它的 `Score`（综合得分）决定。

### 1. 核心奖励维度 (Rewards)
+ `**Alpha**`** (超额收益率)**：对比同期无脑定投（Baseline DCA）的超额利润。这是最核心的进化指标。
+ `**Cumulative Realized PnL**`** (累计已实现盈亏)**：微观引擎在回测中通过批次清算（PositionLot）实际落袋为安的绝对利润。

### 2. 核心惩罚维度 (Penalties - 全悲观原则)
+ **交易摩擦惩罚 (Taker Fee Friction)**：每一次微观成交，必须硬性扣除双边 0.2% 的手续费。**过度频繁交易的基因（如 10 年交易 20000 次）将在此环节被摩擦成本直接抽干，得分为负。**
+ **LOT_SIZE 截断惩罚**：所有回测生成的理论订单，必须经过虚拟交易所的 `stepSize` 和 `minQty` 规范化截断。无法满足步长的订单将被强制丢弃，模拟真实的实盘阻断。
+ **最大回撤惩罚 (Max Drawdown)**：对资金曲线的下行波动进行指数级扣分，淘汰那些靠“死扛”偶尔获利的运气基因。

## 第三篇章：进化纪元与繁衍法则 (Evolutionary Dynamics)
为了方便前端管理进化并决策，每次开启进化任务时，应该有一个大的状态窗口或状态型UI来展示这几层的参数内容，针对不同的策略，这些内容应该可以手动拖拽或按钮调整，确定之后再开启进化。而进化完成后输出的优秀基因，也应该有这样几乎一样的状态窗口来展示这几层的参数内容，以及参数值。

### 1. 顶层：环境 (The Physics Laws / 造物主法则)
+ **边界**：Epoch 内**全员相同**、**基因层不参与改写 Environment**；只描述物理阻断与底线红线。
+ **跨 Epoch 抽样（实现）**：以当前 **冠军包**中的 `Environment` 为均值锚点，对 **DeadReserveRatio**、**GlobalStopLoss** 在各自合法区间内做 **截断正态**（sigma 为区间宽度的 10%）；**MaxLeverage** 在 GA Epoch 中 **固定为 1**（现货）。由进化引擎在跨 Epoch 启动时执行上述抽样。
+ **参数**（当前实现）：
    - **DeadReserveRatio**：死守的现货底仓下限（安全垫），策略层不得突破。
    - **GlobalStopLoss**：全局权益回撤熔断（0 表示关闭，接入产品后生效）。
    - **MaxLeverage**：杠杆上限占位（现货基线为 1；扩展保证金产品时使用）。

### 2. 中层：季节 (The Cyclical Climate)
+ **包含参数**：具有周期性特征、决定宏观节奏，但不需要极其精确的参数。
+ **参数举例**：
    - 资金激进乘数（Aggressiveness Multiplier）  这是中层最核心的进化参数，底层每次买入的资金量倍数，决定了季节。比如激进乘数适中时，代表是春季，底层会进行相对平稳的基础操作， 夏长 、秋收、冬藏总共四季，每个季节暂时确定一个参数，就这个资金激进乘数。这个乘数只随机4次，从低到高代表依次带入到冬、春、夏、秋。
    - 实验室回测与 GA 评估的步进 tick 与 **基准 5 分钟 K 线节拍**对齐；季节层当前主要通过激进乘数等参数施加摩擦，不再用随机 per-season tick 率改变评估频率（旧式随机 tick 与取模跳过曾造成评估步数失真）。
    - 操作的时间偏移（比如，如果每小时操作一次，是在整点操作，还是整点过5分操作） 。但在输出最终种子时，**直接输出 0 偏移（或默认 1m Tick）即可**。实盘由于时间偏移的误差，再也无法摧毁策略，因为它已经在回测里被“抗性训练”过了。
+ **运作逻辑**：在这个 Epoch 中，把 10 年切成 4 段。底层种群在跑这 10 年时，必须忍受这 4 种不同的季节摩擦。
+ **核心规则**：**epoch 内固定、跨 epoch 重采样**；每一季从离散档位按分类分布抽样（categorical），默认对“正常档位=1 or medium”赋更高概率以保证收敛。**运行时约定**：中层（季节）参数在进化完毕后，转化为实盘 SaaS 外壳的【运行时入参（Runtime Arguments）】。**进化实验室**：用四季随机摩擦，确保底层基因不会因为某一种资金乘数而爆仓。**SaaS 决策面**：根据真实的实时大盘水位，从低到高（冬、春、夏、秋）动态抽取档位，喂给这些强悍的底层基因去执行。每一个运行中的交易实例都有一个**调度循环**，云端 **Regime Engine** 判定目前大盘是哪个季节（死水微澜对应冬）。

### 3. 底层：战斗种群基因 (The Combat Genes / 玩家染色体)
+ **包含参数**：**宏观战略**（上节 Macro Genes + Timing）与**微观战术**（`kp, kv, ka`、`MinTradeThreshold`、`SigmoidScale`、`Gamma`、`Beta` 等）同属一条 LunarGenome，一体参与进化。
+ **运作逻辑**：种群在相同的**造物主环境**与**季节带**下跑回测；适应度筛选的是整条染色体，而不是只有微观段。

### 4. 环境切割 (Environment Slicing / 交叉验证)
为了防止基因只适应某一段特定的牛市或熊市，10 年的历史 K 线（含 1m 级微观数据与日线级宏观数据）必须被切割。

+ **训练集 (In-Sample)**：随机抽取多段历史切片组合，作为当前纪元（Epoch）的战场。
+ **坩埚多窗（实现）**：多段滚动窗口（如 all / 5y / 2y / 6m）在各窗上跑回测并与 Ghost DCA 对比，得到加权 Alpha 适应度。**主进化回路中不在每个体、每窗口末执行蒙特卡洛**（实验室默认关闭该环节），避免对每个体、每窗口重复做高成本模拟。
+ **蒙特卡洛终审（实现）**：交易序列的蒙特卡洛模拟 **不进入**适应度打分；仅在 **Epoch 结束**时，对最终挑战者基因追加一次 **1m 全样本复核回测**，并在该次报告中计算路径稳定性（如破产概率、分位收益），写入挑战者/冠军统计档案。若破产概率等指标超过当前告警阈值，引擎记录 **Warn** 日志，**不**改写本次 fitness、**不**阻断 challenger 落库（后续若产品上要“玻璃王冠”降级，可在此层扩展策略）。

### 5. 种群迭代机制 (Population Lifecycle)
在「环境—季节—基因」三层冻结架构下，每一轮进化纪元（Epoch）的外部生存条件都在剧变。种群的生成与迭代必须兼顾「旧日经验」与「全新探索」，严格遵循 **1-4-5 黄金配比法则**（10% / 40% / 50%）。

#### 5.1 初代生成与火种注入 (Initialization & Seed Injection)
+ **10% 旧神火种 (Incumbent Elites)**：自档案库按交易对与模板查询冠军、挑战者等精英记录，按适应度择优**原样轮询注入**；不足时回退到冠军包或策略默认染色体。
+ **40% 伴生变异群 (Targeted Mutants)**：在随机选中的精英拷贝上施加**加大尺度**（如 1.5 倍突变幅度）的加性高斯扰动，覆盖 **整条 LunarGenome**（宏微观字段均可能变动）。
+ **50% 外来移民 (Explorers)（核心机制）**：在合法参数盒内**均匀随机**全新个体，与冠军解耦，用于打破局部最优。

#### 5.2 精英绝对保留 (Elitism)
每一代（Generation）交替时，当前综合 `Score` 排名 `max(1, ceil(PopSize * EliteRatio))` 的个体（`EliteRatio` 默认 **0.05**）**完整克隆**至下一代，不经过交叉与变异。

#### 5.3 正交繁衍 (Orthogonal Crossover)
在父代精英之间做**切片互换**（正交组合），例如：将精英 A 的**动力学感知段**（如 `kp`**、**`kv`，位置与速度权重）与精英 B 的**开火与逃顶段**（如 `ka`、`MinTradeThreshold`），或与 B 的**宏观战略段**进行组合，尝试得到兼具敏锐感知与沉稳开火权、或宏观与微观互补的个体。具体切片边界以实现为准，但语义上须保持「块级」正交，而非逐基因均匀打乱。

#### 5.4 多维破坏性突变 (Multi-dimensional Mutation)
代内子代在交叉后进入**逐基因突变**：每个基因以一定概率（默认约 0.15，可通过突变斜坡上调至上限约 0.55）独立判定是否扰动；扰动为加性高斯步，步长按基因类型设定（例如动力学权重、时间结构整数基因、月相压力等量级不同，以实现为准），最后投影回合法盒。

#### 5.5 挑战者、冠军与归档 (Challenger / Champion / Retired)
+ **Epoch 结束**：最优个体以**挑战者**身份写入档案，**不**自动覆盖冠军。
+ **晋升**：用户在前端对 challenger 执行 **Promote** 后，原冠军标记为**退役**，目标个体成为新冠军，并刷新实例侧冠军缓存（若有）。
+ **实盘种子**：运行中的实例仍只加载**当前冠军**参数包；档案中无冠军时回落到产品内置默认包。

#### 5.6 突变斜坡 (Mutation Ramp)
连续多代无显著改进时，在达到提前终止前优先 **上调** 突变概率与突变幅度（默认放大因子约 1.25，直至各自上限）。

## 第四篇章：回测系统的状态机流转 (State Machine in Backtest)
在回测的每一步策略推进中，系统必须严格模拟实盘的“异步”物理现象：

1. **时钟对齐**：宏观引擎的 Tick（如每小时）与微观引擎的 Tick（如每分钟）在同一条时间轴上交替推进。
2. **库存桥模拟 (Inventory Bridge)**：宏观定投买入的 `DeadBTC` 必须等待微观引擎在侦测到 $ a $（加速度）飙升时，才能被解封为 `FloatBTC` 进行卖出。回测必须精准记录这一库存转移的账本。
